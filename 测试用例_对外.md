# 数据恢复软件 — 测试用例

> 对外版本。不含内部脚本、命令及工程路径。  
> **范围：Windows 10 / Windows 11（64 位）**

---

## 一、测试准备

### 1.1 测试环境

- 主要为 **Windows 10、Windows 11（64 位）**
- **32 位**、**macOS** 本期不测


| OS         | 必测  |
| ---------- | --- |
| Windows 10 | 是   |
| Windows 11 | 是   |


### 1.2 测试机硬件与磁盘规划

**本机（Win10 主力机）示例：**


| 物理磁盘 | 盘符            | 卷标       | 容量                | 用途      | 测试约束          |
| ---- | ------------- | -------- | ----------------- | ------- | ------------- |
| SSD  | **C:**        | —        | 100 GB            | 系统      | 禁止破坏性扫描/删除    |
| SSD  | **D:**        | —        | 约 124 GB          | 数据/导出   | 如 `D:\恢复导出\`  |
| HDD  | **E:**、**F:** | —        | 183 GB / 约 232 GB | 业务数据    | 禁止破坏测试        |
| HDD  | **G:**        | Test5GB  | 5 GB              | 功能测试（小） | xls/ppt 等测试数据 |
| HDD  | **H:**        | Test10GB | 10 GB             | 功能测试（中） | 深度扫描等         |
| HDD  | 未分配           | —        | 约 35 GB           | 预留      | 扩展用           |


**原则：** 破坏性测试仅在 **G:** / **H:**；扫描时选分区，勿选整盘 HDD。

### 1.3 测试数据（G: 盘）


| 路径                      | 内容            |
| ----------------------- | ------------- |
| `G:\recovery-test\xls\` | `.xls` 测试样本   |
| `G:\recovery-test\ppt\` | `.ppt` 测试样本   |
| `D:\恢复导出\`              | 恢复导出目录（测试中建立） |


测试前由测试人员在 G: 盘 **准备好** 上述 xls、ppt 样本；删除类用例后需 **清空回收站** 再扫描。

### 1.4 测前检查


| 步骤  | 操作                        | 通过标准             |
| --- | ------------------------- | ---------------- |
| 1   | 确认系统为 Win10 或 Win11（64 位） | 符合 §1.1          |
| 2   | 打开 G:（及按需 H:）             | 卷标、容量正确，可读写      |
| 3   | 管理员启动恢复软件                 | 设备列表可见 G: / H:   |
| 4   | 确认 G: 测试数据已就绪             | xls、ppt 目录内有样本文件 |
| 5   | 确认 E:/F: 无测试写入            | 业务盘未误操作          |

## 二、测试场景

### A. 安装 & 权限测试


| 用例 ID        | 测试点     | 测试步骤       | 预期结果   |
| ------------ | ------- | ---------- | ------ |
| WIN_INST_001 | 管理员安装   | 右键管理员运行安装包 | 安装成功   |
| WIN_INST_002 | 非管理员安装  | 普通用户安装     | 提示权限不足 |
| WIN_INST_003 | UAC 弹窗  | 安装触发 UAC   | 文案准确   |
| WIN_INST_004 | 安装路径含中文 | 自定义中文路径    | 正常运行   |


### B. 硬盘扫描（通用）


| 用例 ID         | 测试点    | 测试步骤            | 预期结果          |
| ------------- | ------ | --------------- | ------------- |
| DISK_SCAN_001 | 识别测试分区 | 选择 G:（Test5GB）  | 容量、NTFS 显示正确  |
| DISK_SCAN_002 | 快速扫描   | G: 快速/分区扫描完成    | 有结果，无崩溃       |
| DISK_SCAN_003 | 深度扫描   | G: 深度扫描（可选 H:）  | 可完成或正常取消      |
| DISK_SCAN_004 | 扫描范围   | 不选 E:/F: 及整盘物理盘 | 仅 G:（或 H:）被扫描 |
| DISK_SCAN_005 | 类型筛选   | 筛选图片/文档等        | 结果与类型匹配       |


### C. 硬盘扫描 — xls / ppt 稳定加载（P1）


| 用例 ID           | 测试点      | 测试步骤                      | 预期结果                          |
| --------------- | -------- | ------------------------- | ----------------------------- |
| DISK_OFFICE_001 | 扫描测试盘    | 扫描 G:                     | 能列出 `recovery-test` 下 xls、ppt |
| DISK_OFFICE_002 | xls 稳定加载 | 选中 xls → 稳定加载             | 成功，内容可读                       |
| DISK_OFFICE_003 | ppt 稳定加载 | 选中 ppt → 稳定加载             | 成功                            |
| DISK_OFFICE_004 | 删除后预览    | 删除部分样本 → 清回收站 → 再扫 → 稳定加载 | 成功；失败为缺陷                      |
| DISK_OFFICE_005 | 导出校验     | 导出到 D: → 本地 Office/WPS 打开 | 文件正常                          |


> `文件片段 @ 0x…`、类型误显 DOC 等视为磁盘残留，**不作为本缺陷判定依据**。
