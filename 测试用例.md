# 数据恢复软件 — 测试用例

> **范围：** Windows 10 / Windows 11（64 位）。32 位、macOS 本期不测。

---

## 一、测试准备

### 1.1 测试环境


| OS         | 必测  | 说明              |
| ---------- | --- | --------------- |
| Windows 10 | 是   | 当前台式 **主力测试机**  |
| Windows 11 | 是   | 其它机器上测，磁盘规划与上一致 |


### 1.2 测试机硬件与磁盘规划


| 物理磁盘      | 盘符            | 卷标       | 容量                | 用途      | 测试约束         |
| --------- | ------------- | -------- | ----------------- | ------- | ------------ |
| SSD（磁盘 1） | **C:**        | —        | 100 GB            | 系统      | 禁止破坏性扫描/删除   |
| 同上        | **D:**        | —        | 约 124 GB          | 数据      | —            |
| HDD（磁盘 0） | **E:**、**F:** | —        | 183 GB / 约 232 GB | 业务数据    | 禁止破坏测试       |
| 同上        | **G:**        | Test5GB  | 5 GB              | 功能测试（小） | xls/ppt 测试数据 |
| 同上        | **H:**        | Test10GB | 10 GB             | 功能测试（中） | 深度扫描等        |
| 同上        | 未分配           | —        | 约 35 GB           | 预留      | 扩展分区         |


**原则：**

1. 破坏性测试仅在 **G:**（主）或 **H:**（按需）。
2. **C/D/E/F** 仅安装、导出、业务，不批量删除测试样本。
3. 扫描时选分区 **G: / H:**，勿选整盘 HDD（约 466GB），避免扫到 E/F。

### 1.3 测试数据（G: 盘）


| 路径                      | 内容          |
| ----------------------- | ----------- |
| `G:\recovery-test\xls\` | `.xls` 测试样本 |
| `G:\recovery-test\ppt\` | `.ppt` 测试样本 |


测试前在 G: 盘准备好 xls、ppt 样本；删除后清空回收站再扫描。恢复软件建议管理员身份运行。

## 附录（内部 · 不对外分发）

测试数据脚本与命令（点击展开）

**生成测试数据：**

```powershell
pip install xlwt
cd D:\数据恢复Ritridata\scripts
python gen_test_office.py
```

**删除子集（DISK_OFFICE_004 前）：**

```powershell
Remove-Item "G:\recovery-test\xls\测试报表_01*.xls","G:\recovery-test\xls\测试报表_02*.xls","G:\recovery-test\xls\测试报表_03*.xls" -Force
Remove-Item "G:\recovery-test\ppt\下载样本_01.ppt","G:\recovery-test\ppt\下载样本_02.ppt" -Force
Clear-RecycleBin -Force -ErrorAction SilentlyContinue
```

**重置：** 再次执行 `python gen_test_office.py`。

可选：离线 ppt 放入 `scripts\ppt_templates\`；图片放入 `G:\recovery-test\assets\`。
